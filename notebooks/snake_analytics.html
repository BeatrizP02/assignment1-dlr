Setup and Data Loading

In this section we import libraries, set up paths, and load the CSV logs for each Snake model:

  • A2C_Efficiency
  • PPO_Efficiency
  • PPO_Maximizer

These CSVs contain training metrics like episode score, episode length, and death type. These metrics are gathered and saved during the initial training.
We also define a helper function for calculating a rolling mean, which smooths noisy results to better show long-term learning trends.

In[88]:
paths = {
    "A2C_Efficiency": LOGS / "training_metrics_snake_a2c_efficiency.csv",
    "PPO_Efficiency": LOGS / "training_metrics_snake_ppo_efficiency.csv",
    "PPO_Maximizer":  LOGS / "training_metrics_snake_ppo_maximizer.csv",
}

dfs = {}
for name, p in paths.items():
    if not p.exists():
        raise FileNotFoundError(f"Missing: {p}")
    df = pd.read_csv(p)
    df.columns = [c.lower() for c in df.columns]
    # keep only the basics + death_type if available
    keep = [c for c in ["episode","timestep","score","episode_length","death_type"] if c in df.columns]
    df = df[keep].sort_values("episode").reset_index(drop=True)
    # ensure numeric
    for col in ["episode","timestep","score","episode_length"]:
        if col in df.columns:
            df[col] = pd.to_numeric(df[col], errors="coerce")
    dfs[name] = df.dropna(subset=["episode","timestep","score","episode_length"])

def rolling_mean(s, window=20):
    return s.rolling(window, min_periods=max(5, window//4)).mean()

Learning Curve: Score

This plot shows the rolling average score across training timesteps.
It demonstrates how each agent improves over time. PPO typically converges faster and to a higher score, while A2C improves at a slower rate.
The efficiency persona tends to score higher because it prioritizes steady growth, while the maximizer persona is more aggressive and only prioritizes eating apples.

Rolling Crash Rates: Wall vs Self

This analysis looks at the cause of agent deaths over time.
We track whether the snake crashes into the wall or into itself.
Patterns reveal differences in play style:

  • Efficiency persona may drift toward wall crashes by staying alive longer but getting cornered.
  • Maximizer persona may crash into itself more often as it aggressively grows.

The take away from these graphs are A2C never really learns to avoid the walls, this could be why it has the worst performance. When looking at PPO Efficiency vs Maximizer we can see that PPO_Efficiency learns to avoid the walls a bit sooner when compared to PPO_Maximizer.

window = 40

for name, df in dfs.items():
    if "death_type" not in df.columns or df["death_type"].isna().all():
        print(f"[{name}] No 'death_type' info  skipping crash-rate plot.")
        continue

    # Rolling proportion of episodes ending by 'wall' or 'self'
    is_wall = (df["death_type"] == "wall").astype(float)
    is_self = (df["death_type"] == "self").astype(float)

    wall_share = is_wall.rolling(window, min_periods=5).mean()
    self_share = is_self.rolling(window, min_periods=5).mean()

    plt.figure()
    plt.plot(df["timestep"], wall_share, label="Wall (rolling share)")
    plt.plot(df["timestep"], self_share, label="Self (rolling share)")
    plt.ylim(0, 1)
    plt.xlabel("Timesteps")
    plt.ylabel("Share of episodes")
    plt.title(f"Crash Rates  {name} (window={window})")
    plt.legend()
    plt.show()

eval_paths = {
    "A2C_Efficiency": LOGS / "eval_snake_a2c_efficiency.csv",
    "PPO_Efficiency": LOGS / "eval_snake_ppo_efficiency.csv",
    "PPO_Maximizer":  LOGS / "eval_snake_ppo_maximizer.csv",
}

eval_dfs = {}
for name, p in eval_paths.items():
    if not p.exists():
        print(f"Missing eval file: {p}")
        continue
    df = pd.read_csv(p)
    df.columns = [c.lower() for c in df.columns]
    eval_dfs[name] = df
